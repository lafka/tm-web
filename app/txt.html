A Tiny Philosophy

The user matches the signature provided by server with it's client key, this to verify that no changes have been made to the request, if verification succeeds the client can assume normal operation.

Subsequent requests will be signed by a combination of client key, shared secret, and the data. The above procedure requires a attack to have first hand knowledge of both client key and the shared secret before a new request can be made.
